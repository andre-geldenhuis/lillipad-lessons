Blink an LED

Let’s start by blinking one of the white LED’s on the lilipad breakout board.

Arduino code is written in a form of C++. This code has 2 minimum required parts, a setup section and a loop section.

void setup(){
}

void loop(){

}

The above code doesn’t do anything but it the bare minimum. Lets do something more interesting.

void setup(){
    pinMode(5,OUTPUT); //Sets pin 5 as an output.
}

void loop() {
    digitalWrite(5, HIGH); //Function to turn output 5 on
    delay(1000); //wait 1000 ms or 1 second
    digitalWrite(5,LOW); //Function to turn output 5 off
    delay(1000);
}

Try this code out, what happens?

Note that the code after // doesn’t run, it is just a comment to make the code easier to follow. digitalWrite and delay are functions which take an input, in this case 5 and 1000 and do something with them.

Variables

What if we wanted to light up the second LED (pin 6) instead of the first one? We would have to replace the number 5 three times. It is easy to make mistakes when replacing the number over and over. Instead we can create a variable to refer to the led pin, that way we only have to change one number.

int ledpin = 5; // Set the variable ledpin to 5. We also specify that it is 
                //a integer ie a whole number like 1 2 or 100, but not 1.2 

void setup(){
    pinMode(ledpin,OUTPUT); //Sets pin 5 as an output.
}

void loop() {
    digitalWrite(ledpin, HIGH); //Function to turn output 5 on
    delay(1000); //wait 1000 ms or 1 second
    digitalWrite(ledpin,LOW); //Function to turn output 5 off
    delay(1000);
}
  • Try blink the led of A4 witha period of half a second.
  • Try replacing the duration 1000 with a variable of your own.

Functions

What if we wanted to turn LEDS on in a sequence? We could copy paste the earlier code over and over and put in a different number for each LED. That would be a lot of copy pasting, so we can create our own functions, just like delay and digitalWrite that can combine a section of code so we can write less code!


void myBlink(int ledpin, int duration){
    pinMode(ledpin,OUTPUT); //We can also set the pinMode here if we like. 
    digitalWrite(ledpin, HIGH); //Function to turn output ledpin on
    delay(duration); //wait duration ms
    digitalWrite(ledpin,LOW); //Function to turn output ledpin off
    delay(duration);
}

void setup(){} //We don't need to do anything here at the moment.

void loop(){
    myBlink(A2,500);
}

Here we create function called myBlink it has two inputs, a pin to blink and a duration to blink. Inside this function the variables ledpin and duration are temporarily created, so we need to define their type. Here we make them both integers.

The void at the start of the function definition just lets the compiler know that this function doesn’t return any values.

Notice that the setup() loop() and sections look a lot like our function! This is because they are functions too. setup() gets called by the arduino when it starts up then loop() is run over and over once it is going.

Using the buzzer and for loops

We can use the buzzer with handy functions called tone() and noTone(). The buzzer is connected to pin 7 on the breakout board.


void setup() {

}

void loop() {
  // turn off tone function for pin 7:
  noTone(7);
  // play a 440Hz tone (standard pitch A) on pin 7 for 200 ms:
  tone(7, 440, 200);
  delay(200);

  // turn off tone function for pin 7:
  noTone(7);
  // play a 494Hz tone (a B note on a piano) on pin 7 for 500 ms:
  tone(7, 494, 500);
  delay(500);

  // turn off tone function for pin 7:
  noTone(7);
  // play a 523Hz tone (a C on a piano) on pin 7 for 500 ms:
  tone(7, 523, 300);
  delay(300);
}

For this example we would have had to look up what frequency corresponds to which note and it would involve lots of copy pasting to write a more complicated melody. Helpfully the arduino programming environment helpfully comes with a lot of handy examples. Navigate to File -> Examples -> Digital -> toneMelody

It will produce the following: 

/*
  Melody

 Plays a melody

 circuit:
 * 8-ohm speaker on digital pin 8

 created 21 Jan 2010
 modified 30 Aug 2011
 by Tom Igoe

This example code is in the public domain.

 http://www.arduino.cc/en/Tutorial/Tone

 */
#include "pitches.h"

// notes in the melody:
int melody[] = {
  NOTE_C4, NOTE_G3, NOTE_G3, NOTE_A3, NOTE_G3, 0, NOTE_B3, NOTE_C4
};

// note durations: 4 = quarter note, 8 = eighth note, etc.:
int noteDurations[] = {
  4, 8, 8, 4, 4, 4, 4, 4
};

void setup() {
  // iterate over the notes of the melody:
  for (int thisNote = 0; thisNote < 8; thisNote++) {

    // to calculate the note duration, take one second
    // divided by the note type.
    //e.g. quarter note = 1000 / 4, eighth note = 1000/8, etc.
    int noteDuration = 1000 / noteDurations[thisNote];
    tone(8, melody[thisNote], noteDuration);

    // to distinguish the notes, set a minimum time between them.
    // the note's duration + 30% seems to work well:
    int pauseBetweenNotes = noteDuration * 1.30;
    delay(pauseBetweenNotes);
    // stop the tone playing:
    noTone(8);
  }
}

void loop() {
  // no need to repeat the melody.
}

There is a lot more going on in this example! Let’s break it down.

#include "pitches.h" includes the file pitches.h, you can see it’s contents in the tab at the top of the editor window. It defines tones like #define NOTE_C4 as the frequency 262 which is what the tone() function expects. Incidentally this is how A2 is defined as pin 16 as we saw in the led chaser example.

The following defines an array called noteDurations.

int noteDurations[] = {
  4, 8, 8, 4, 4, 4, 4, 4
};

This is just a way to have a collection of numbers, in this case integers, defined in one place. One nice thing about arrays, is that they are easy to iterate over with a loop.

  for (int thisNote = 0; thisNote < 8; thisNote++) {
    int tempnote=noteDurations[thisNote];
  }

the for(.....){ } structure repeats the code inside the { } for a number of times. The number of times we will repeat the code is determined by what’s in side the
for( ) brackets.
for( start counter; test if counter has reached condition; increment-counter)

  • The counter is a variable like any other, so we have to give it a type int thisNote=0
    • We also set the variable to 0 initially
  • We check if the counter variable has reached a given condition thisNote < 8. In this case, the loop will keep running as long as thisNote is less than 8
  • Each time thought the loop we increase the value of thisNote by one. The ++ operator is a shorthand for this

Aside — debugging with print statements

Google how to print to the serial console with an arduino. Try to print out the tempnote value and see how it changes as the loop runs. Here is the partial code to get you started. It doesn’t have any print statements, you’ll have to find out how to use those yourself. 

int noteDurations[] = {
  4, 8, 8, 4, 4, 4, 4, 4
};

void setup() {
    Serial.begin(9600);  //start the serial communication at 9600 baud.
    for (int thisNote = 0; thisNote < 8; thisNote++) {
        int tempnote=noteDurations[thisNote];
        //Serial print goes here
    }
}

void loop() {
  // no need for anything here.
}
